## 🎯 プロジェクト概要

### システム名
**コンプライアンス学習システム (sgcompliance)**

### 対象範囲
- ユーザー数：60～100名
- 学習コンテンツ配信
- テスト実施・採点
- 進捗管理
- 修了証・バッジ発行
- 管理機能

### 技術スタック
- **Frontend**：Next.js 14 + React + TypeScript + Tailwind CSS
- **Backend**：Supabase (PostgreSQL + Auth)
- **Storage**：Supabase Storage
- **Deployment**：Vercel

### デザイン指針
- **色**：白基調 + #0d3280（コーポレートカラー）
- **トーン**：シンプル・ミニマル・プロフェッショナル

## 🔑 主要決定事項

### バッジ仕様
- **種類**：全体合格バッジのみ（1種類）
- **取得条件**：全コース合格（80点以上）
- **有効期限**：1年間（更新制）

### 修了証仕様
- **記載内容**：ユーザー名 + コース名 + 発行日 + シリアルナンバー
- **形式**：PDF
- **表示方法**：ユーザーはサイト表示 / 管理者はリスト表示

### テスト仕様
- **最大挑戦回数**：3回
- **合格ライン**：80点
- **採点**：リアルタイム
- **試行履歴**：全試行を保持

### 動画コンテンツ
- **フォーマット**：YouTube埋め込み
- **構成**：動画 + テスト（セット）
- **有効化条件**：90%以上再生でテスト有効化

## 📊 データベース設計

### 主要テーブル
- `users` - ユーザー情報
- `departments` - 部門
- `courses` - コース
- `lessons` - 学習単元
- `questions` - クイズ問題
- `quiz_attempts` - テスト実施記録（最大3試行）
- `certificates` - 修了証
- `badges` - バッジ定義
- `user_badges` - ユーザーバッジ（1年有効期限付き）
- `video_watch_progress` - 動画再生進捗

### セキュリティ
- RLS (Row Level Security) で権限管理
- ユーザーは自分のデータのみ表示
- 管理者は全データ表示可能

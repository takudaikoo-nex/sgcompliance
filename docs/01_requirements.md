# コンプライアンス学習システム - 要件定義

**作成日：2026-06-18**

## プロジェクト概要
- 新規構築（Supabase、GitHub、Vercel新設）
- 対象ユーザー：60～100名
- 参考プロジェクト：AI試験くん（同様構成）

## デザイン
- 白基調 + #0d3280（コーポレートカラー）
- 従来デザインから大きく刷新

## 学習コンテンツ
- テキスト・Markdown
- 画像・図解
- YouTube埋め込み動画
- クイズ・インタラクティブ

## テスト・クイズ
- リアルタイム採点
- CSV形式で質問/回答をSupabase保存
- SQLクエリ + API連携
- 複数回チャレンジ対応予定（段階的改修）
- 合格ライン：80点

## 修了証・バッジ
- 管理画面で確認可能
- 種類は企画書作成後に修正、まずは提案

## ユーザーロール
1. 管理権限ユーザー
   - 部門別に進捗・結果確認
   - 管理画面へのアクセス
2. 受講者ユーザー
   - 自分の進捗確認
   - コンテンツ受講・テスト実施

## 動画
- YouTube埋め込み
- 本数少ないため、別途テスト受講可能にする
- **動画+テストをセットで提供**

## 技術スタック
- Next.js + Supabase + Tailwind CSS + TypeScript（前プロジェクト同様）

## 保存・参考資料
- Notionの.mdファイル：企画書作成、コンテンツ生成用

## 詳細決定事項

### バッジ
- 全体合格バッジのみ
- 1年更新（毎年取得条件リセット）

### 修了証
- 記載情報：ユーザー名
- 表示方法：ユーザーはサイト表示 / 管理者はリスト表示

### チャレンジ仕様
- 最大3回まで
- 過去ログを細かく確認可（全試行結果を保持）

### 部門管理
- 管理者は全部門のデータを見られる（部門ごとの見える化）

### 動画テスト
- 動画+テストをセット提供
